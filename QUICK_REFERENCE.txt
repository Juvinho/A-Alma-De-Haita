🚀 QUICK START (5 MINUTOS)
═════════════════════════════════════════════════════════════════════════

1. Instalar
   ────────
   # Windows
   install_sprites_deps.bat
   
   # Linux/macOS
   chmod +x install_sprites_deps.sh && ./install_sprites_deps.sh
   
   # Manual
   pip install -r requirements_sprites.txt


2. Processar
   ──────────
   python sprite_processor.py --input ./raw_assets/


3. Copiar
   ───────
   cp -r sprites_processed/* public/sprites/


4. Usar no React
   ──────────────
   <Sprite character="enygma" expression="happy" />


5. Deploy
   ──────
   npm run build && npm run deploy

⚙️ CONFIGURAÇÃO
═════════════════════════════════════════════════════════════════════════

No topo de sprite_processor.py:

CANVAS_WIDTH = 600              # Largura do canvas (pixels)
CANVAS_HEIGHT = 1080            # Altura do canvas (pixels)
REMBG_MODEL = "isnet-anime"     # Modelo IA (anime otimizado)
OUTPUT_FOLDER = "sprites_processed"
SUPPORTED_EXTS = {".png", ".jpg", ".jpeg", ".webp"}

DEFAULT_NAMES = [
    "normal", "happy", "thinking", "sad",
    "angry", "surprised", "surprised_2", "crying", "smug"
]

💡 DICAS
═════════════════════════════════════════════════════════════════════════

Rápido?
  → Use --no-process se fundo já transparente

Lento?
  → pip install rembg[gpu] onnxruntime-gpu

Sprite pequeno?
  → Aumentar CANVAS_HEIGHT / CANVAS_WIDTH

Fundo não sai?
  → Usar ferramenta manual ou aumentar REMBG_MODEL qualidade

Múltiplos personagens?
  → Processar em pastas separadas com --output different


📈 DEPOIS DO DEPLOY
═════════════════════════════════════════════════════════════════════════

Novo personagem?
  1. Adicionar PNGs em raw_assets/
  2. python sprite_processor.py --input ./raw_assets/
  3. cp -r sprites_processed/* public/sprites/
  4. Pronto!

Melhorar qualidade?
  1. Aumentar resolução PNGs originais
  2. Ajustar REMBG_MODEL ou CANVAS_*
  3. Reprocessar: python sprite_processor.py --overwrite

Otimizar performance?
  1. Usar em cache
  2. Lazy load se necessário
  3. Imagemin para comprimir
